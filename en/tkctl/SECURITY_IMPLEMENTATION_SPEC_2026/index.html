KISA Vulnerability Analysis & Assessment Implementation Specification (2026)

This document defines the security scan and remediation logic that tkctl will perform at a production-grade level of detail, based on the Detailed Guide for Technical Vulnerability Analysis and Assessment of Critical Information and Communication Infrastructure.

1. Architecture and Data Model​

The security module is designed with an interface-based architecture for extensibility. All scan results must be serializable to JSON.

1.1 Data Structures (Go Structs)​

package security

// CheckStatus - Scan status constants
type CheckStatus string

const (
    StatusGood  CheckStatus = "GOOD"  // Compliant
    StatusBad   CheckStatus = "BAD"   // Vulnerable
    StatusNA    CheckStatus = "NA"    // Not Applicable (service not in use, etc.)
    StatusError CheckStatus = "ERROR" // Check failed (insufficient permissions, etc.)
)

// SecurityItem - Scan item metadata
type SecurityItem struct {
    Code        string      // e.g., "U-01"
    Category    string      // "Linux", "DBMS", "Web"
    Level       string      // "High", "Medium", "Low"
    Name        string      // Item name (e.g., "Restrict remote access to root account")
    Description string      // Detailed description
    Criteria    string      // Assessment criteria
    Guide       string      // Remediation guide (text)
}

// CheckResult - Scan result
type CheckResult struct {
    Item       SecurityItem
    Status     CheckStatus
    CurrentVal string      // Current state (e.g., "PermitRootLogin yes")
    Message    string      // Detailed message
    Fixed      bool        // Whether remediation was performed
    FixLog     string      // Remediation result log
}

// Scanner interface
type Scanner interface {
    Name() string
    Detect(ctx context.Context) ([]CheckResult, error)
    Fix(ctx context.Context, itemCode string) (bool, error)
}

2. Unix/Linux Server Checks (U-01 ~ U-30)​

Target: RHEL/CentOS/Rocky, Ubuntu/Debian families

CodeItem NameSeverityCheck DetailsFix Logic
U-01Restrict remote access to root accountHighWhether SSH root login is allowedsed -i 's/^PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config then reload
U-02Password complexity settingsHighCombination of letters/numbers/special characters and lengthInject settings such as minlen=8, lcredit=-1 in /etc/security/pwquality.conf
U-03Account lockout threshold settingsHighAccount lockout after failed logins (5 attempts)Check and add pam_faillock.so or pam_tally2.so configuration
U-04Password file protectionHigh/etc/passwd permission checkchmod 644 /etc/passwd && chown root:root /etc/passwd
U-05root home/path directory permissionsHighWhether PATH contains .Guide to remove . from PATH variable in user .bash_profile, etc. (auto-fix risky)
U-06File/directory owner settingsHighExistence of files without owner (nouser)Scan with find / -nouser -o -nogroup and report (manual deletion recommended)
U-07passwd file owner/permissionsHigh/etc/passwd permissionsSame as U-04 (duplicate item, separate check)
U-08shadow file owner/permissionsHigh/etc/shadow permissionschmod 400 /etc/shadow && chown root:root /etc/shadow
U-09hosts file owner/permissionsHigh/etc/hosts permissionschmod 600 /etc/hosts && chown root:root /etc/hosts
U-10xinetd.conf file permissionsHigh/etc/xinetd.conf permissionschmod 600 /etc/xinetd.conf (if file exists)
U-11rsyslog.conf file permissionsHigh/etc/rsyslog.conf permissionschmod 640 /etc/rsyslog.conf
U-12services file permissionsHigh/etc/services permissionschmod 644 /etc/services
U-13SUID, SGID configured filesHighUnnecessary SetUID filesProvide guide to remove SUID (chmod -s) for non-essential files
U-14User startup file permissionsHigh.bashrc etc. permissionsGuide to set home directory config file permissions to 640 or below
U-15World Writable filesHighFiles writable by anyoneScan with find / -type f -perm -2 and report
U-16Directories without device filesHighUnnecessary files in /devScan for regular files within /dev directory
U-17$HOME/.rhosts permissionsHighr-command authentication fileRecommend file deletion (rm .rhosts)
U-18Connection IP and port restrictionHighhosts.allow/deny settingsCheck if ALL:ALL is set in /etc/hosts.deny
U-19Disable Finger serviceHighWhether Finger service is runningsystemctl disable --now finger
U-20Disable Anonymous FTPHighAnonymous FTP access allowedSet anonymous_enable=NO in vsftpd.conf
U-21Disable r-series servicesHighrlogin, rsh, rexecStop related services with systemctl disable --now rlogin, etc.
U-22Cron file owner/permissionsHigh/etc/crontab etc. permissionschmod 640 /etc/crontab
U-23Services vulnerable to DoS attacksHighecho, discard, etc.Set disable = yes for those services in xinetd configuration
U-24Disable NFS serviceHighUnnecessary NFS runningStop with systemctl stop nfs if not in use
U-25NFS access controlHigh/etc/exports settingsVulnerable if * (allow all) setting exists
U-26Disable automountdHighAuto-mount servicesystemctl disable --now autofs
U-27RPC service checkHighUnnecessary RPC servicesAnalyze rpcinfo -p results
U-28NIS, NIS+ checkHighNIS service usagesystemctl disable --now ypserv
U-29Disable tftp, talk servicesHighVulnerable service statussystemctl disable --now tftp
U-30Sendmail version checkHighWhether latest version is usedCheck version via package manager

3. DBMS (MariaDB/MySQL) Checks (D-01 ~ D-10)​

Connection method: Connect to local/remote DB as root account and execute SQL queries.

CodeItem NameSeverityCheck Query / LogicFix Guide
D-01Change default account passwordHighCheck if root password is NULL or defaultProvide ALTER USER statement
D-02Remove unnecessary accountsMediumCheck for default accounts such as test, guestDROP USER 'test'@'%'
D-03Restrict remote accessHighCheck bind-address and accounts with Host='%'Local binding and host privilege reduction
D-04Password policy settingsHighvalidate_password plugin configuration valuesGuide for INSTALL PLUGIN validate_password
D-05Login failure lockoutHighCheck connection_control pluginPlugin activation and threshold configuration
D-06File system access restrictionHighCheck local_infile variableSET GLOBAL local_infile = 0
D-07Enable audit loggingHighCheck general_log or Audit pluginEnable audit logging (my.cnf)
D-08Separation of administrator privilegesHighCheck non-admin accounts with SUPER privilegeRevoke unnecessary admin privileges (REVOKE)
D-09Session timeout settingsMediuminteractive_timeout, wait_timeout valuesRecommend timeout value of 600 seconds (10 minutes) or less
D-10Apply latest security patchesHighCompare SELECT VERSION() resultsRecommend updating to latest minor version

4. Web Server (Nginx) Checks (N-01 ~ N-10)​

Target: Main configuration (nginx.conf) and sites-enabled/*.conf parsing.

CodeItem NameSeverityCheck LogicFix Configuration
N-01Remove directory listingHighDetect autoindex on; directiveChange to or delete autoindex off;
N-02Restrict unnecessary methodsHighCheck for missing limit_except blockAdd limit_except GET POST { deny all; }
N-03Restrict admin page accessHighCheck ACL for paths like /admin, /managerSet allow 127.0.0.1; deny all;
N-04Restrict hidden file accessHighCheck for location ~ /\. { deny all; }Add the location block
N-05Remove unnecessary filesHighSearch for .bak, .old, .log files in web rootRecommend moving backup files outside web root
N-06Hide web service informationHighCheck for server_tokens on;Set server_tokens off;
N-07Error page configurationMediumCheck default error page exposureConnect custom pages via error_page directive
N-08File upload size limitHighCheck client_max_body_size valueSet appropriate size limit (e.g., 10M)
N-09Web process privilegesHighCheck if user directive is rootRun as www-data or nginx account
N-10HTTPS encrypted communicationHighCheck SSL/TLS configuration (listen 443 ssl)Recommend applying certificate and redirecting port 80

5. Detailed Implementation Guidelines​

5.1 Scan Execution Flow​

  1. OS/Service Detection: Detect whether the current system is Linux and which services (MariaDB, Nginx) are running.
  2. Parallel Scanning: When possible, perform parallel scans by category using goroutines.
  3. Result Aggregation: Collect all results into a list of CheckResult structs.
  4. Report Generation: Save results as a JSON file (scan_report_YYYYMMDD.json).

5.2 Backup and Rollback (Safety First)​

  • Backup: Must perform cp source dest.bak before modifying configuration files.
  • Rollback: If service restart fails after remediation, implement automatic restoration from backup file (Rollback() method).

5.3 CLI Command Design​

tkctl analyze security              # Full scan
tkctl analyze security --target os  # OS (Linux) scan only
tkctl analyze security --target db  # DB scan only
tkctl fix security --code U-01      # Fix specific item
tkctl fix security --auto           # Fix all auto-remediable items (use with caution)

This specification is designed to meet the robustness of a production environment and the KISA certification standards.
