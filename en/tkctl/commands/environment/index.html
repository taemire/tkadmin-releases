3.5 Environment Configuration (env)​

CommandFunctionKey Flags (Options)
(default)View all environment configuration status--set : Automatically apply recommended values for each item
selinuxSELinux configuration[mode] : enforcing/permissive/disabled
--set : Apply configuration
ulimitSystem limits configuration[value] : Open Files limit value (e.g., 65535)
--set : Apply configuration
firewallFirewall port configuration[port] : Port to open
--set : Apply configuration (includes defaults like SSH, 443)
sshSSH port configuration[port] : Port number to change to
--set : Restart service and sync with firewall
webWeb port configuration[port] : Port number to change to
--set : Restart service and sync with firewall
dbDB port configuration[port] : Port number to change to
--set : Apply configuration
--yes : Auto-approve service restart
localeSystem Locale configuration[lang] : Language code (ko/en)
--set : Apply configuration
service-limitService LimitNOFILE configuration[service] [value] : Service name + file descriptor limit
Alias: svc-limit

Diagnoses and configures the OS environment (SELinux, Ulimit, Firewall, SSH) optimized for running the Tachyon solution. Use the tkctl env [subcommand] format for detailed control of each item. Running tkctl env without arguments performs a summary check of all items.

The --set (or -s) option is used to actually apply configurations. When arguments are omitted, the TACHYON recommended values appropriate for each environment are automatically selected.

3.5.1 SELinux Configuration (selinux)​

Checks and optimizes the current SELinux runtime mode and permanent configuration.

Usage:

# Check status
tkctl env selinux

# Automatically set to recommended value (disabled)
tkctl env selinux --set

# Set to a specific mode
tkctl env selinux disabled --set
tkctl env selinux permissive --set

Features:

  • Skips redundant operations if already set to the target mode.
  • When setting disabled mode, changes the permanent configuration and applies permissive at runtime for immediate effect until reboot.

Output Example​

2025-12-24 21:03:14 [INFO] ### SELinux Configuration ###
--------------------------------------------
 Item                  Status
--------------------------------------------
 Runtime               Disabled
 Permanent             disabled
--------------------------------------------
2025-12-24 21:03:14 [SUCC] Environment check complete.

3.5.2 Limits Configuration (ulimit)​

Diagnoses the system-wide (/etc/security/limits.conf) and per-service LimitNOFILE configuration status.

Usage:

# Check limits status
tkctl env ulimit

# Automatically set system limit to recommended value (65535)
tkctl env ulimit --set

# Set system limit to a specific value
tkctl env ulimit 99999 --set

Note: Applying system limit settings requires logging out and back in or a system restart.

3.5.3 Firewall Configuration (firewall)​

Checks the system firewall (firewalld) status and opens ports required for TACHYON operation.

Usage:

# Diagnose firewall status and open ports
tkctl env firewall

# Automatically open TACHYON default ports (SSH, 443)
tkctl env firewall --set

# Open a specific additional port
tkctl env firewall 8080 --set

Configuration logic:

  • No arguments: Automatically detects the current system's SSH port and opens it along with port 443.
  • Port specified: Opens the specified port with the tcp protocol.

Output Example​

2025-12-24 21:03:35 [INFO] ### Firewall Configuration Diagnostics ###

 [ Firewall Status ]
--------------------------------------------
 Service/Zone          Status
--------------------------------------------
 firewalld             Inactive
 iptables              Inactive
--------------------------------------------
2025-12-24 21:03:35 [SUCC] Environment check complete.

3.5.4 SSH Port Configuration (ssh)​

Checks and changes the OpenSSH service port number.

Usage:

# Check current SSH port
tkctl env ssh

# Change SSH port to 2022
tkctl env ssh 2022 --set

# Revert SSH port to default (22)
tkctl env ssh --set

Automatic configuration notes:

  • Firewall integration: When changing the port via --set, the new port is automatically allowed in the firewall. (Prevents connection blocking)
  • Automatic service restart: After configuration changes, the sshd service is automatically restarted for immediate effect.

Output Example​

2025-12-24 21:03:46 [INFO] ### SSH (OpenSSH) Configuration Diagnostics ###

 [ ### SSH (OpenSSH) Configuration Diagnostics ### ]
--------------------------------------------
 Item                  SSH Port
--------------------------------------------
 OpenSSH                22
--------------------------------------------

 Use '--set <port>' to change the SSH port
2025-12-24 21:03:46 [SUCC] Environment check complete.

3.5.5 Web Port Configuration (web)​

Checks and changes the listening port of the TACHYON web service (Nginx HTTPS).

Usage:

# Check current web port
tkctl env web

# Change HTTPS port to 8443
tkctl env web 8443 --set

# Revert to default (443)
tkctl env web --set

v0.4.20+ Extended Options:

OptionDescription
--mode unified/separatedSwitch between unified/separated mode
--console <port>Admin console port in separated mode (default: 8443)
--backend <port>Backend API port in separated mode (default: 443)
--dry-runPreview changes (no actual application)
--restoreRestore nginx.conf from backup

Extended usage examples:

# Switch to separated mode
tkctl env web --mode separated --set

# Separated mode + specify ports
tkctl env web --mode separated --console 9443 --backend 10443 --set

# Preview (check without changes)
tkctl env web --mode separated --dry-run

# Restore from backup
tkctl env web --restore

Detailed behavior:

  • Nginx configuration change: Finds listen ... ssl directives in /etc/nginx/nginx.conf or conf.d/*.conf and changes the port.
  • Firewall integration: Immediately allows the changed port in the firewall.
  • Service control: Automatically restarts the Nginx service to apply changes.
  • Automatic backup: Automatically backs up nginx.conf before configuration changes (.bak).
  • Restore feature: Use the --restore option to restore settings from the last backup.

3.5.6 DB Port Configuration (db)​

Changes the MariaDB database connection port and batch-synchronizes all TACHYON services and management tools that reference it.

Usage:

# Check current DB port
tkctl env db

# Change MariaDB port to 13306 (auto-approve)
tkctl env db 13306 --set --yes

# Revert to default (3306)
tkctl env db 3306 --set

Automation scope and impact: This command performs extensive system synchronization beyond a simple port change.

  1. System port change: Changes the MariaDB port in /etc/my.cnf.d/server.cnf.
  2. Batch service reference update:
    • Searches all service configuration files (*.yml_dev) within TACHYON_HOME and replaces the port in jdbc:mariadb:// connection strings.
    • Also updates the db_port entry in the global configuration (app_info.properties) to maintain system-wide consistency.
  3. Firewall and restart:
    • Registers the changed port in the firewall.
    • Sequentially restarts all affected services including MariaDB, API, Manager, Watchdog, and tkadmin.
    • Since service downtime occurs during operation, user approval is requested unless the --yes option is provided.

Caution (Nginx Stream): If Nginx uses a stream block (at the bottom of nginx.conf) to proxy the DB port (such as 3306), the port in this section is not automatically changed by default.

v0.4.20+ Nginx Stream Auto-Sync:

OptionDescription
--sync-streamAutomatically update nginx stream upstream port
--dry-runPreview changes (no actual application)
# Change DB port + auto-sync nginx stream
tkctl env db 3307 --set --sync-stream

# Preview (check without changes)
tkctl env db 3307 --dry-run

Using the --sync-stream option will also change the port in the upstream tachyon_mariadb block of nginx.conf.

Output example:

2026-01-08 16:30:00 [INFO] Updating SERVICE configurations (TACHYON_HOME)...
2026-01-08 16:30:01 [INFO]   Updated: api.yml_dev
2026-01-08 16:30:01 [INFO]   Updated: manager.yml_dev
...
2026-01-08 16:30:02 [INFO] Updating nginx stream upstream 'tachyon_mariadb' to port 13306...

3.5.7 DB Port Detection and Connection Troubleshooting​

tkctl automatically detects the port when connecting to MariaDB in the following order:

  1. MariaDB configuration file: The port value specified in the [mysqld] section of my.cnf or server.cnf.
  2. Nginx Stream configuration: The Nginx upstream tachyon_mariadb port synchronized via tkctl env db --sync-stream.
  3. Default: 3306.

Problem scenario: Changed DB port but tkctl cannot connect

  1. Symptom: dial tcp 127.0.0.1:3306: connect: connection refused error occurs.
  2. Cause: The port is not specified in the MariaDB configuration file and is not synchronized with the Nginx configuration, so the default (3306) is attempted.
  3. Solution:
    • Method A (recommended): Run the tkctl env db <PORT> --sync-stream command to update the Nginx configuration, and tkctl will detect and connect to the correct port.
    • Method B: Explicitly specify port=<PORT> in the [mysqld] section of the MariaDB configuration file (server.cnf, etc.).

Tip: The tkctl env db command synchronizes the Nginx Stream configuration simultaneously with the port change, making it the safest way to change the port.

3.5.8 Locale Configuration (locale)​

Checks and changes the system Locale (language and character set) configuration.

Background: When installing RockyLinux 9 in Non-GUI (Minimal) mode, the default is set to LANG=C or LANG=en_US, which may cause Korean characters to display incorrectly in the product console. This can be resolved by changing to ko_KR.UTF-8 or en_US.UTF-8.

Usage:

# Check current locale status
tkctl env locale

# Set to Korean locale
tkctl env locale ko --set

# Set to English locale
tkctl env locale en --set

Features:

  • If the current setting is already *.UTF-8, it skips without making changes.
  • If the language pack (glibc-langpack-ko) is not installed, manual installation instructions are displayed.
  • Air-gapped environment support: Outputs only a warning without automatic package installation.

Output example:

2026-01-21 22:52:34 [INFO] ### Locale (Language/Character Set) Configuration ###
-------------------------------------------------
 Item                  Status
-------------------------------------------------
 System Locale         LANG=ko_KR.UTF-8
 VC Keymap             kr
 Korean Language Pack  ✓ Installed
-------------------------------------------------
2026-01-21 22:52:34 [SUCC] Environment check complete.

When language pack is not installed:

2026-01-21 22:00:00 [WARN] Language pack (glibc-langpack-ko) is not installed.
  ℹ Install via ISO file or local repository:
    rpm -ivh glibc-langpack-ko-*.rpm

3.5.9 Service LimitNOFILE Configuration (service-limit)​

Checks and changes the systemd LimitNOFILE value for individual services.

Usage:

# View LimitNOFILE status for all services
tkctl env service-limit

# Change LimitNOFILE for a specific service
tkctl env service-limit nginx 65535
tkctl env service-limit mariadb 131072

# Alias (svc-limit) also available
tkctl env svc-limit redis 65535

Detailed behavior:

  • Writes [Service]\nLimitNOFILE=<value> to /etc/systemd/system/<service>.service.d/override.conf
  • Automatically performs systemctl daemon-reload + systemctl restart <service>
  • Allowed range: 1024 ~ 1048576

Allowed services: mariadb, redis, zookeeper, kafka, opensearch, logstash-kafka-os, nginx, opensearch-dashboards, tkadmin

Tab auto-completion:

  • First argument: Service name auto-completion
  • Second argument: Suggested values (4096, 8192, 16384, 32768, 65535, 131072)

Output Example (Status Check)​

2026-03-07 14:00:00 [INFO] ### Service LimitNOFILE Status ###
-----------------------------------------------------------------------
 Service Name                    Limit (NOFILE)        Status
-----------------------------------------------------------------------
 mariadb                          32768                  OK
 redis                            524288                 OK
 nginx                            65535                  OK
 opensearch                       65535                  OK
 tkadmin                          65535                  OK
-----------------------------------------------------------------------

Output Example (Value Change)​

2026-03-07 14:00:00 [SUCC] nginx LimitNOFILE = 65535 configuration complete

3.5.10 Nginx ACL Configuration (web acl)​

Manages the Nginx web server's Access Control List (ACL) to restrict admin console access to specific IPs only. Supports IP addresses (IPv4) or CIDR ranges (e.g., 192.168.1.0/24).

Scope of impact: ACL controls access to the admin console (frontend) only. It does not affect the backend API service (port 443), so API calls and service integrations are not affected.

Usage:

# 1. Check ACL policy
tkctl env web acl list

# 2. Add administrator IP (required)
tkctl env web acl add 203.0.113.10
tkctl env web acl add 192.168.100.0/24

# 3. Enable ACL (blocks all connections, only allowed IPs can access)
# Caution: Always add administrator IPs (add) before enabling (enable).
tkctl env web acl enable

# [Other]
# Remove a specific allowed IP
tkctl env web acl remove 203.0.113.10

# Disable ACL (allow all IP access)
tkctl env web acl disable

# Test configuration (syntax check and applied status verification)
tkctl env web acl test

Cautions:

  • When ACL is enabled (enable), all connections are blocked (deny all) by default, and only IPs added with the add command are allowed.
  • Localhost (127.0.0.1) is not automatically included in the allow rules for internal communication. Explicit addition is recommended depending on the location structure (explicit addition is recommended in the current implementation).
  • After configuration changes (add, remove, enable, disable), Nginx must be restarted for changes to take effect.
    • tkctl service restart nginx

Configuration file behavior:

  • By default, injects allow/deny directives into the http or location block of /app/nginx/conf/nginx.conf (or the installation path).
  • The --config option allows you to directly specify the target configuration file path.
