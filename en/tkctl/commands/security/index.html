3.6 Security Analysis & Remediation​

tkctl provides integrated security vulnerability analysis (analyze) across OS, Web (Nginx, etc.), and DB (MariaDB) areas, as well as automated remediation (fix) for discovered vulnerabilities.

3.6.1 Security Analysis (Analyze Security)​

Inspects the current system configuration state according to KISA Critical Information Infrastructure Vulnerability Analysis criteria and generates a report.

Usage​

tkctl analyze security [flags]

Key Flags​

FlagDescriptionDefault
--target, -tSpecify inspection target (os, web, db, all)all
--format, -fSpecify report format (json, html)json
--output, -oFile path to save resultsAuto-generated
--quiet, -qSuppress console output (for automation scripts)false

Usage Examples​

1. Full system analysis with HTML report generation

tkctl analyze security --format html
# Result: tkctl-security-report-20260118-143000.html generated

Open the generated HTML file in a browser to view intuitive inspection results distinguished by color codes.

2. Quick OS configuration check only

tkctl analyze security -t os

3. CI/CD pipeline integration (JSON output)

tkctl analyze security --format json --quiet
# Result: tkctl-security-report-YYYYMMDD-HHMMSS.json generated (no logs)

3.6.2 Security Remediation (Fix Security)​

Automatically modifies settings for items found to be 'Vulnerable' in the analysis results to strengthen security.

Caution: Security remediation directly modifies system settings. It is strongly recommended to perform a backup (--backup=true) and preview changes with --dry-run mode first.

Usage​

tkctl fix security [flags]

Key Flags​

FlagDescriptionDefault
--idSpecific vulnerability ID to remediate (e.g., U-01)Required (except when using --report or --all)
--reportAutomated remediation based on analysis report (JSON)""
--allBatch remediation of all supported items (use with caution)false
--dry-runSimulate remediation without actual changesfalse
--backupBackup original configuration files before remediationtrue
--interactive, -iInteractive mode requesting confirmation for each itemfalse

Usage Examples​

1. Simulate remediation for a specific vulnerability (U-01) with dry-run

tkctl fix security --id U-01 --dry-run
# Result:
# [INFO] Security remediation process started
# [INFO] [U-01] Remediation started: Setting PermitRootLogin to 'no' in SSH config file (sshd_config) and restarting service.
# [INFO]   [DryRun] No actual changes will be made.
# [SUCC] All remediation processes completed.

2. Actual remediation for a specific vulnerability (U-01)

tkctl fix security --id U-01
# Result:
# [INFO] [U-01] Remediation started: Setting PermitRootLogin to 'no' in SSH config file (sshd_config) and restarting service.
# [SUCC]   Backup complete: /etc/ssh/sshd_config.bak_20260118_194500
# [SUCC]   Remediation complete

3. Batch remediation based on report (--report)

Selects and remediates only vulnerable (VULNERABLE) items based on analysis results (tkctl analyze --format json).

tkctl fix security --report tkctl-security-report-20260118.json
# Result:
# [INFO] Analyzing report: tkctl-security-report-20260118.json
# Vulnerabilities found: 15 | Auto-remediable: 3
# ... (remediation process continues)

4. Remediation in interactive mode (--interactive)

Requests user confirmation before each item. Options: y (Yes), n (No/Skip), a (All/Yes for all remaining), q (Quit/Abort)

tkctl fix security --all --interactive
# Result:
# [INFO] 85 remediation items selected.
# ----------------------------------------------------------------
# [INFO] [U-01] Remediation target: Restrict root remote access (SSH PermitRootLogin setting)
# [U-01] Proceed with remediation? [y(Yes)/n(No)/a(All)/q(Quit)]: y
# [INFO]   -> Starting remediation...
# [SUCC]   Backup complete: /etc/ssh/sshd_config.bak_20260119_143000
# [SUCC]   Remediation complete
# ...

3.6.4 Troubleshooting​

Q. I want to revert after remediation caused an issue. A. The fix command always creates a backup (*.bak) of the original file during execution. Overwrite the original with the backup file or manually restore the contents. Example: cp /etc/ssh/sshd_config.bak_20260118 /etc/ssh/sshd_config
