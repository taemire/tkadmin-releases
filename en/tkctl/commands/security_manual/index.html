3.7 Security Vulnerability Manual Inspection Guide​

Automated Analysis Notice (v0.4.24+)

Starting from v0.4.24, automated analysis and remediation for all 85 KISA 2026 Critical Information Infrastructure technical vulnerability items has been implemented. Therefore, most items in this manual can be automatically processed with the tkctl analyze security command, and one-click remediation is available via tkctl fix security.

For detailed implementation specifications, refer to the KISA 2026 Security Vulnerability Implementation Specification. This document is maintained as a reference for legacy environments or special situations where automated analysis is not possible.

When tkctl analyze security results occasionally show a 'Manual inspection required (MANUAL)' status or when cross-verification is needed, refer to this guide.

Note: This document is based on the KISA Critical Information Infrastructure Technical Vulnerability Analysis and Assessment Detailed Guide.

3.7.1 OS Security Inspection (U-01 ~ U-73)​

U-01. Root Remote Access Restriction​

ItemDetails
CategoryAccount Management
Risk LevelHigh
Inspection TargetRemote access services such as SSH, Telnet

Inspection Method

# Check SSH configuration
grep -i "PermitRootLogin" /etc/ssh/sshd_config

# Check Telnet configuration (if in use)
cat /etc/securetty

Compliant Criteria

  • PermitRootLogin no is set

Remediation Method

# Change SSH configuration
sudo sed -i 's/^#*PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

U-02. Password Complexity Settings​

ItemDetails
CategoryAccount Management
Risk LevelHigh
Inspection TargetPAM configuration

Inspection Method

# Check PAM complexity configuration
cat /etc/pam.d/system-auth | grep pam_pwquality
cat /etc/security/pwquality.conf

Compliant Criteria

  • Combination of alphabetic, numeric, and special characters
  • Minimum 8 characters

Remediation Method

# /etc/security/pwquality.conf settings
minlen = 8
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1

U-03. Account Lockout Threshold Settings​

ItemDetails
CategoryAccount Management
Risk LevelHigh
Inspection TargetPAM faillock configuration

Inspection Method

grep -E "deny=|faillock" /etc/pam.d/system-auth
grep -E "deny=|faillock" /etc/pam.d/password-auth

Compliant Criteria

  • Account lockout after 5 or fewer login failures

Remediation Method

# faillock configuration example
auth required pam_faillock.so preauth deny=5 unlock_time=600
auth [default=die] pam_faillock.so authfail deny=5 unlock_time=600

U-04. Password File Protection​

ItemDetails
CategoryAccount Management
Risk LevelHigh
Inspection Target/etc/passwd, /etc/shadow

Inspection Method

# Check whether Shadow file is in use
awk -F: '$2 != "x" && $2 != "!" && $2 != "*" {print $1}' /etc/passwd

Compliant Criteria

  • All account passwords are encrypted and stored in /etc/shadow

U-05. No UID 0 Except Root​

ItemDetails
CategoryAccount Management
Risk LevelHigh
Inspection Target/etc/passwd

Inspection Method

awk -F: '$3 == 0 {print $1}' /etc/passwd

Compliant Criteria

  • No accounts with UID 0 other than root

U-06. Root Home/Path Permissions​

ItemDetails
CategoryFile Permissions
Risk LevelHigh
Inspection Targetsu command permissions

Inspection Method

ls -l /usr/bin/su
cat /etc/pam.d/su | grep pam_wheel

Compliant Criteria

  • su command is restricted to the wheel group

3.7.2 WEB Security Inspection (N-01 ~ N-06)​

Nginx web server security configuration inspection items.

Nginx Configuration File Locations​

# Common paths
/etc/nginx/nginx.conf
/etc/nginx/conf.d/*.conf

# Tachyon TTS environment
/usr/local/TACHYON/TTS40/nginx/conf/nginx.conf

N-01. Version Information Disclosure Restriction​

ItemDetails
CategoryInformation Disclosure
Risk LevelMedium

Inspection Method

grep -i "server_tokens" /etc/nginx/nginx.conf
curl -I http://localhost 2>&1 | grep Server

Compliant Criteria

  • server_tokens off; is set

Remediation Method

# Inside the http block of nginx.conf
http {
    server_tokens off;
}

N-02. HTTP Method Restriction​

ItemDetails
CategoryAccess Control
Risk LevelMedium

Inspection Method

grep -i "limit_except" /etc/nginx/nginx.conf
curl -X OPTIONS http://localhost -I

Compliant Criteria

  • Unnecessary methods (PUT, DELETE, TRACE, etc.) are blocked

Remediation Method

location / {
    limit_except GET POST {
        deny all;
    }
}

N-03. Directory Listing Restriction​

ItemDetails
CategoryAccess Control
Risk LevelHigh

Inspection Method

grep -i "autoindex" /etc/nginx/nginx.conf

Compliant Criteria

  • autoindex off; or no setting (default is off)

Remediation Method

# Remove autoindex on; or change to off
autoindex off;

N-04. File Upload Restriction​

ItemDetails
CategoryConfiguration Management
Risk LevelMedium

Inspection Method

grep -i "client_max_body_size" /etc/nginx/nginx.conf

Compliant Criteria

  • Limited to an appropriate size (e.g., 10M)

Remediation Method

http {
    client_max_body_size 10M;
}

N-05. Hidden File Access Restriction​

ItemDetails
CategoryAccess Control
Risk LevelMedium

Inspection Method

grep -E "location.*\\\\." /etc/nginx/nginx.conf
curl http://localhost/.htaccess

Compliant Criteria

  • Access to files starting with . is blocked

Remediation Method

location ~ /\. {
    deny all;
}

N-06. Log Configuration Check​

ItemDetails
CategoryLog Management
Risk LevelMedium

Inspection Method

grep -E "access_log|error_log" /etc/nginx/nginx.conf
ls -l /var/log/nginx/

Compliant Criteria

  • Both access_log and error_log are enabled

Remediation Method

http {
    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log warn;
}

3.7.3 DB Security Inspection (D-01 ~ D-06)​

MariaDB/MySQL database security configuration inspection items.

MariaDB Configuration File Locations​

# Common paths
/etc/my.cnf
/etc/mysql/my.cnf
/etc/my.cnf.d/*.cnf

# Tachyon TTS environment
/usr/local/TACHYON/TTS40/mariadb/my.cnf

D-01. Default Account Management​

ItemDetails
CategoryAccount Management
Risk LevelHigh

Inspection Method

-- Check for anonymous accounts
SELECT User, Host FROM mysql.user WHERE User='';

-- Check for test databases
SHOW DATABASES LIKE 'test%';

Compliant Criteria

  • No anonymous accounts or test databases exist

Remediation Method

-- Delete anonymous accounts
DROP USER ''@'localhost';
DROP USER ''@'%';

-- Delete test database
DROP DATABASE IF EXISTS test;

-- Or run mysql_secure_installation

D-02. Root Remote Access Restriction​

ItemDetails
CategoryAccount Management
Risk LevelHigh

Inspection Method

SELECT User, Host FROM mysql.user WHERE User='root';

Compliant Criteria

  • root account Host allows only 'localhost' or '127.0.0.1'

Remediation Method

-- Remove remote root access
DELETE FROM mysql.user WHERE User='root' AND Host NOT IN ('localhost', '127.0.0.1', '::1');
FLUSH PRIVILEGES;

D-03. Configuration File Permissions​

ItemDetails
CategoryConfiguration Management
Risk LevelMedium

Inspection Method

ls -l /etc/my.cnf
ls -l /var/lib/mysql/

Compliant Criteria

  • Configuration file: 640 or less (owned by root/mysql)
  • Data directory: 750 (owned by mysql)

Remediation Method

chmod 640 /etc/my.cnf
chown root:mysql /etc/my.cnf
chmod 750 /var/lib/mysql
chown mysql:mysql /var/lib/mysql

D-04. Password Complexity​

ItemDetails
CategoryPassword
Risk LevelMedium

Inspection Method

SHOW VARIABLES LIKE 'validate_password%';
-- Or
SHOW VARIABLES LIKE 'simple_password_check%';

Compliant Criteria

  • Password validation plugin is enabled

Remediation Method

-- MariaDB
INSTALL PLUGIN simple_password_check SONAME 'simple_password_check';

-- MySQL
INSTALL COMPONENT 'file://component_validate_password';
SET GLOBAL validate_password.policy = MEDIUM;

D-05. Network Binding Configuration​

ItemDetails
CategoryNetwork
Risk LevelHigh

Inspection Method

grep bind-address /etc/my.cnf /etc/my.cnf.d/*
netstat -tlnp | grep 3306

Compliant Criteria

  • bind-address = 127.0.0.1 or restricted to a specific IP

Remediation Method

# my.cnf [mysqld] section
[mysqld]
bind-address = 127.0.0.1

D-06. Audit Log Configuration​

ItemDetails
CategoryLog Management
Risk LevelMedium

Inspection Method

grep -E "log_error|server_audit" /etc/my.cnf
ls -l /var/log/mysql/

Compliant Criteria

  • Error log is enabled
  • Audit plugin usage is recommended

Remediation Method

# my.cnf [mysqld] section
[mysqld]
log_error = /var/log/mysql/error.log

# Audit plugin (optional)
plugin-load = server_audit=server_audit.so
server_audit_logging = ON
server_audit_file_path = /var/log/mysql/audit.log
