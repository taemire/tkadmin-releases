3.1 Basic System Management​

This section covers the basic commands (version, info, admin, completion, update) for checking and maintaining the overall system status.

3.1.1 Version Information (version)​

CommandFunctionKey Flags (Options)
(default)View detailed system and component versionsNone

Checks version information of TKCTL and related middleware. Provides detailed version information of all TACHYON components and middleware in table format, and detailed descriptions can be viewed via --help.

Usage​

tkctl version [flags]

3.1.2 System Information (info)​

CommandFunctionKey Flags (Options)
(default)View system resources and disk usageNone

Provides an overview of license information, system resources, disk usage, and service status. Detailed information is automatically collected and displayed from the /usr/local/TACHYON path and related middleware data.

Usage​

tkctl info [flags]

Output Example​

  OpenSearch                      15G         /usr/local/TACHYON/TTS40/opensearch/opensearch/data

2025-12-24 21:02:28 [SUCC] Service status check completed.

Note: OS Memory Over-Utilization Information The memory usage reported by tkctl info includes OS-level physical memory utilization. Especially in Linux environments, idle memory is used as Page Cache/Buffer for performance optimization, so the displayed usage may appear higher than the actual process usage — this is known as "over-utilization." This is not a system failure but a normal result of OS performance management. For detailed technical explanation, refer to [Appendix A - 8. OS Memory Management Specification].

3.1.3 Administrator Account Management (admin)​

The admin command is a subcommand system for managing TACHYON administrator accounts.

SubcommandFunctionArguments
reset [user_id]Reset administrator passwordTarget user ID
reset-ip [user_id]Reset administrator IP ACLTarget user ID
superuser -P <password>Change tkadmin superuser password-P : New password

reset — Password Reset​

Resets the password of an administrator account.

Usage​
tkctl admin reset <USER_ID>
Reset Password​

!TumsAdmin2025

Examples​
# Reset admin account password
tkctl admin reset admin

# Reset specific user password
tkctl admin reset user123
Output Example​
2025-12-07 16:15:00[INFO] Updating password for user ID 'admin'...
2025-12-07 16:15:01[SUCC] Password update for user 'admin' completed.
2025-12-07 16:15:01[INFO] New password and key have been set.

reset-ip — IP ACL Reset​

Clears the IP access control (ACL) settings for a TACHYON administrator account. Use this command when an account is locked out due to IP ACL restrictions.

Usage​
tkctl admin reset-ip <USER_ID>
Examples​
# Clear IP ACL for tsadmin account
tkctl admin reset-ip tsadmin
Output Example​
2026-03-30 10:00:00[INFO] Resetting IP ACL for user 'tsadmin'...
2026-03-30 10:00:00[INFO] Current IP ACL: allow_ip1=192.168.1.10, allow_ip2=(none)
2026-03-30 10:00:01[SUCC] IP ACL for user 'tsadmin' has been reset.
info

Changes are applied to both 000TACHYON.TPU_WCONSOLE_USERS (global) and the tenant table.

3.1.4 Auto-Completion (completion)​

CommandFunctionKey Flags (Options)
(default)Generate shell auto-completion scripts[bash/zsh/fish/powershell]

Generates auto-completion scripts for various shells.

Supported Shells​

  • Bash
  • Zsh
  • Fish

Usage​

tkctl completion [bash|zsh|fish|powershell]

Installation​

Bash (Linux):

# System-wide
tkctl completion bash | sudo tee /etc/bash_completion.d/tkctl

# Per-user
tkctl completion bash > ~/.local/share/bash-completion/completions/tkctl

Zsh:

# Enable completion (first time only)
echo "autoload -U compinit; compinit" >> ~/.zshrc

# Install completion script
tkctl completion zsh > "${fpath[1]}/_tkctl"

Fish:

tkctl completion fish > ~/.config/fish/completions/tkctl.fish

v0.4.20+ Enhanced Tab Completion​

Starting from v0.4.20, all file path completions are disabled, and intelligent auto-completion features have been significantly enhanced.

Static Completion:

Suggests predefined values immediately for applied flags.

FlagApplied CommandsCompletion Values
--formatanalyze, backupjson, csv, table
--levelGlobaldebug, info, warn, error
--targetservice, fixall, tachyon, middleware
--shellcompletionbash, zsh, fish
--statusanalyze securityvulnerable, good, manual
--sourceanalyzeopensearch, mariadb, filesystem

Dynamic Completion:

Queries real-time index/field lists through OpenSearch connections to provide suggestions.

FlagApplied CommandsBehavior
--indexbackup opensearch, analyzeReal-time index list query
--fieldanalyze media-top, etc.Field list query for selected index
--patternbackup opensearch ilmIndex pattern suggestion
--policybackup opensearch ilmILM policy list query
--aliasbackup opensearchAlias list query

Usage Examples:

# Static completion
tkctl analyze --format <Tab>
# json  csv  table

# Dynamic completion (requires OpenSearch connection)
tkctl backup opensearch index --index <Tab>
# tachyon-agent-2026.01  tachyon-media-2026.01  ...

3.1.5 Service Update (update)​

CommandFunctionKey Flags (Options)
nginxNginx update-f : Package file path (required)
--force : Skip version check and force proceed
redisRedis update-f : Package file path (required)
opensearchOpenSearch update-f : Package file path (required)
--force : Skip version check and force proceed

Updates the key middleware services (Nginx, Redis, OpenSearch) of the TACHYON system. Rather than simple file replacement, a version verification system is built in to prevent system failures caused by incorrect version installations.

Usage​

tkctl update [service] [flags]

Options​

  • --file, -f: Specifies the path to the update package file (.tar.gz).
  • --force: Ignores version safety checks and forces the update to proceed. (Useful in downgrade scenarios)

Safety Guards​

  1. Downgrade Prevention: Automatically stops the update if the package version is lower than the currently installed version.
  2. Major Version Protection: Outputs a warning when the first digit (Major) of the version differs (e.g., v2 -> v3) to raise awareness of compatibility issues in advance.
  3. Integrity Check: After decompression, directly executes the binary to extract and verify version information.

Example (Detailed Log)​

$ tkctl update nginx -f nginx-1.29.4.tar.gz

2024-12-21 15:30:10 [INFO] Starting nginx update...
2024-12-21 15:30:10 [INFO] Version verification: Current 1.29.3 -> Target 1.29.4 (safe)
...
2024-12-21 15:30:15 [SUCC] nginx update completed.

3.1.6 Deployment Package Generation (update package) [DEPRECATED]​

Generates middleware packages for deployment to production environments (air-gapped networks, etc.). Downloads the latest source from an internet-connected environment, compiles binaries optimized for the production server specifications, and repackages them.

Usage​

tkctl update package [service]

Supported Services​

  • nginx: Downloads latest stable source and includes OpenSSL/PCRE2/Zlib static compilation.
  • redis: Downloads and compiles the latest source.
  • opensearch: Repackages based on official binary.
  • mariadb: Downloads binary tarball via MariaDB REST API (LTS 10.11 default). Supports archive.mariadb.org fallback when REST API fails.

Build Dependency Auto-Check (Build Dependency Guard)​

Automatically checks for required build tools on the system before packaging.

  1. Checks: gcc, make, perl (including FindBin module), tar, curl
  2. Auto-Install: If running with root privileges, automatically installs missing tools via dnf or apt.
  3. Manual Install Guide: If privileges are insufficient or auto-install is not possible, displays optimized installation commands for the system.
$ tkctl update package nginx

2025-12-23 11:00:05 [INFO] Checking build dependencies...
2025-12-23 11:00:05 [WARN] Missing dependencies found: gcc, perl
2025-12-23 11:00:05 [INFO] Automatically installing required packages...
...
2025-12-23 11:02:40 [SUCC] Build dependency installation complete. Continuing with packaging.

Normal Upgrade Scenario

$ tkctl update opensearch --file opensearch-2.19.4.tar.gz

Output:

3.1.7 Deployment Package Generation (update package)​

CommandFunctionKey Flags (Options)
nginxGenerate Nginx package--version, --clean-cache
redisGenerate Redis package--version, --clean-cache
opensearchDownload OpenSearch package--version, --clean-cache
mariadbDownload MariaDB package--version, --clean-cache

Flags​

FlagShortDescription
--version-VSpecify a particular version (e.g., --version 10.11.11)
--clean-cache-Clean cache directory before build

Provides expert functionality for professional engineers to download, compile, and package the latest middleware versions for air-gapped environment deployment. This feature tracks the currently installed major version to maintain compatibility with the production environment.

Features​

  • Major Version Protection: If currently using v1.x, even if v2.x is released, it automatically finds and packages the latest patch version in the v1.x series.
  • Automatic Dependency Resolution: Automatically downloads libraries such as PCRE2, OpenSSL, and Zlib needed for Nginx builds along with the source for static compilation.
  • Standardized Filenames: Generates packages with version-explicit filenames like nginx-1.29.4.tar.gz.

Examples​

# Generate Nginx latest patch package
tkctl update package nginx

# Output:
2025-12-20 12:12:40[INFO] Checking internet connection...
2025-12-20 12:12:41[SUCC] Internet connection confirmed.
2025-12-20 12:12:41[INFO] Currently installed version: 1.29.3 (major: 1)
2025-12-20 12:12:41[INFO] Detecting latest stable version for nginx (filtering major 1)...
2025-12-20 12:12:42[INFO] Target version (1.29.4) is newer than current (1.29.3). Package update is recommended.
2025-12-20 12:12:43[INFO]   - Downloading dependency: pcre2-10.45
2025-12-20 12:12:48[INFO]   - Configuring: --with-pcre=../pcre2-10.45 ...
2025-12-20 12:12:53[INFO]   - Compiling...
2025-12-20 12:14:25[SUCC] Package generated successfully: /root/dist/packages/nginx-1.29.4.tar.gz
# Generate MariaDB latest LTS package (auto-detect when version not specified)
tkctl update package mariadb

# Output:
2026-02-09 16:59:51[INFO] Checking internet connection...
2026-02-09 16:59:52[SUCC] Internet connection confirmed.
2026-02-09 16:59:52[INFO] Currently installed version: 10.11.2 (major: 10.11)
2026-02-09 16:59:52[INFO] Detecting latest stable version for mariadb (filtering major 10.11)...
2026-02-09 16:59:52[INFO] Detecting latest version in MariaDB 10.11 series...
2026-02-09 16:59:54[INFO] Target version (10.11.16) is newer than current (10.11.2). Package update is recommended.
2026-02-09 16:59:54[INFO] Downloading mariadb 10.11.16 source/artifacts...
2026-02-09 17:01:12[INFO] Building and packaging mariadb 10.11.16...
2026-02-09 17:01:20[SUCC] Package generated successfully: /root/dist/packages/mariadb-10.11.16.tar.gz

Automatic Version Detection (v0.5.7): When no version is specified, it automatically extracts the major series of the currently installed MariaDB (e.g., 10.11), and detects the latest patch version via REST API (downloads.mariadb.org) or archive (archive.mariadb.org) fallback.

# Specify a particular version (--version flag)
tkctl update package mariadb --version 10.11.10
tkctl update package nginx -V 1.24.0

# Generate packages for all services
tkctl update package all

Notes​

  • This command must be executed in an environment with active internet connection (e.g., development server, internet-accessible production server).
  • Generated packages can be copied to air-gapped servers and installed using the tkctl update command.

Build Environment Requirements​

Nginx and Redis package generation requires source code compilation, so the following build tools must be installed on the system:

ComponentDescriptionRHEL/Rocky/Alma PackageUbuntu/Debian Package
GCCC/C++ compilergcc, gcc-c++build-essential
MakeBuild automation toolmakemake
PerlRequired for OpenSSL buildperl-core, perl-IPC-Cmdperl
TarArchive handlingtartar

Note: OpenSearch and MariaDB download pre-built binaries, so compilation tools are not required.

Automatic Dependency Installation​

tkctl automatically checks build dependencies before packaging and attempts automatic installation if missing packages are detected.

How It Works:

  1. Auto-detects the OS distribution (RHEL, Rocky, Ubuntu, etc.).
  2. Identifies the package manager (dnf, yum, apt).
  3. When running as root, automatically installs missing packages.
  4. If auto-installation is not possible, provides manual installation commands.

Output Example (Auto Dependency Installation):

2025-12-23 17:00:00[INFO] Checking build dependencies...
2025-12-23 17:00:00[WARN] Missing build dependencies: gcc, perl
2025-12-23 17:00:00[INFO] Automatically installing missing packages...
2025-12-23 17:00:00[INFO] Installing packages via dnf: gcc, gcc-c++, perl-core, perl-IPC-Cmd
...
2025-12-23 17:00:15[SUCC] Build dependency installation completed.

Manual Installation (When Auto-Install Fails):

# RHEL 9 / Rocky Linux 9 / AlmaLinux 9
sudo dnf install -y gcc gcc-c++ make perl-core perl-IPC-Cmd

# Ubuntu / Debian
sudo apt-get install -y build-essential make perl
