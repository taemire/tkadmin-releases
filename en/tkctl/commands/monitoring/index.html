3.3 Monitoring & Analysis (monitoring)​

3.3.1 Capacity Check (size)​

CommandFunctionKey Flags (Options)
(default)Query top N MariaDB table sizes[count] : Number of results (default 20)
tablesDetailed MariaDB table size query[count] : Number of results
--by-month : View monthly data aggregation
opensearchOpenSearch index size query--by-month : View monthly data aggregation

Checks the capacity of MariaDB tables and OpenSearch indices. Two subcommands are provided: size tables and size opensearch, each supporting detailed sorting and aggregation options.

Usage​

# Basic usage (check top n MariaDB tables)
tkctl size [n]

# Detailed MariaDB table check
tkctl size tables [n] [flags]

# Detailed OpenSearch index check
tkctl size opensearch [flags]

MariaDB Table Size​

# All tables
tkctl size tables

# Top 20 tables
tkctl size tables 20

# Backward compatibility
tkctl size 20

# LOG table monthly data aggregation
tkctl size tables --by-month
tkctl size db -m

Output example:

Key features:

  • MySQL-style tabular output format
  • Automatic unit conversion (B/KB/MB/GB)
  • Thousands separator formatting (14,005)
  • Descending sort by size
  • Dynamic column width adjustment
  • Monthly aggregation option (-m, --by-month): Automatically detects date columns in *_LOG tables and aggregates data by month

# Full index list
tkctl size opensearch

# Monthly aggregation (date-patterned indices)
tkctl size opensearch -m
tkctl size opensearch --by-month

3.3.2 Log Analysis (analyze)​

CommandFunctionKey Flags (Options)
recommendData-driven analysis recommendations--month : Target month for analysis (YYYY-MM)
media-topMedia control process/path Top N--month : Target month
--limit : Number of results (default 20)
--output : Save to CSV file
agent-anomalyDetect agents with excessive logging--month : Target month
--threshold : Threshold multiplier vs. average (default 5.0)
process-topPer-process log aggregation--month : Target month
--limit : Number of results
db-trendDB data growth trend analysis--output : Save to CSV file

Analyzes log data from MariaDB and OpenSearch for root cause diagnosis and anomaly detection. Helps identify the cause of log surges in production environments and automatically recommends items that need analysis.

Usage​

tkctl analyze [subcommand] [flags]

Subcommands​

SubcommandDescriptionData Source
media-topMedia control process+path Top N analysisOpenSearch
agent-anomalyDetect agents with excessive loggingOpenSearch
process-topPer-process log aggregation Top NOpenSearch
recommendData-driven analysis recommendationsMariaDB + OpenSearch
db-trendDB data growth trend analysisMariaDB

Common Flags​

FlagShortDescriptionDefault
--month-mTarget month for analysis (YYYY-MM)Current month
--limit-nLimit number of results20
--timeoutQuery timeout (seconds)30
--output-oFile path to save results (.csv, .txt)(console output)

Analysis Recommendations (recommend)​

Automatically diagnoses the data state and recommends which analyses should be performed. Provides an automated answer to the operator's question: "What should I analyze?"

Usage:

# Default execution (based on current month)
tkctl analyze recommend

# Analysis based on a specific month
tkctl analyze recommend --month 2026-01

Output example:

   Current Month: 55,941  |  Previous Month: 28,970  |  Growth Rate: +93.1%

🔍 Summary: 1 item requires analysis, 2 items normal

Assessment criteria:

  • 🔴 Surge (Priority 1): 100% or more increase over previous month - immediate analysis required
  • ⚠️ Rapid Increase (Priority 2): 50-100% increase over previous month - analysis recommended
  • ✅ Normal (Priority 3): Less than 50% change from previous month

Media Control Log Analysis (media-top)​

Analyzes the Top N occurrence frequency by process+path combination in media control event logs. Helps identify which processes accessed which paths most frequently.

Usage:

# Top 10 for the current month
tkctl analyze media-top --limit 10

# Analysis for a specific month
tkctl analyze media-top --month 2026-01

# Save results to CSV file
tkctl analyze media-top --month 2026-01 --output media_report.csv

Agent Anomaly Detection (agent-anomaly)​

Detects agents that are logging excessively compared to the average. Helps identify cases where abnormal volumes of events are generated from specific PCs.

Usage:

# Default execution (detect agents logging 5x or more above average)
tkctl analyze agent-anomaly

# Adjust threshold (detect only agents logging 10x or more above average)
tkctl analyze agent-anomaly --threshold 10

# Analysis for a specific month
tkctl analyze agent-anomaly --month 2026-01

Output example:

Process Aggregation Analysis (process-top)​

Aggregates log occurrence counts by process and displays the top N. Helps identify which processes are generating the most events.

Usage:

# Top 20 processes
tkctl analyze process-top

# Display only top 5
tkctl analyze process-top --limit 5

# Analysis for a specific month and save to file
tkctl analyze process-top --month 2026-01 --output process.csv

DB Growth Trend Analysis (db-trend)​

Analyzes the growth trends of DB and log data to support storage capacity planning. Two modes are available: Fast Mode (default) and Precise Mode.

Usage:

# Default execution (Fast Mode: instant analysis based on information_schema)
tkctl analyze db-trend

# Precise analysis (Precise Mode: detailed analysis based on COUNT(*) queries, takes time)
tkctl analyze db-trend --precise
tkctl analyze db-trend -P

# Save results to CSV file
tkctl analyze db-trend --output trend.csv

1. Fast Mode (Default)​

Uses information_schema metadata to return results instantly (O(1)). Does not impose load on the running DB and provides an integrated capacity report for the entire system (MariaDB + OpenSearch + Disk).

Output example:

Key features:

  1. Integrated capacity report: View MariaDB, OpenSearch, install directory, and disk availability at a glance
  2. Capacity Planning: Estimate monthly/yearly growth and predict disk saturation based on current data size
  3. Top 50 large tables: Detailed display of top 10 tables by size with summary of the rest

2. Precise Mode (Detailed Analysis)​

Activated with the --precise or -P option. Performs actual COUNT(*) queries to analyze exact row counts and growth trends by period (1 week, 1 month, 1 quarter, 1 year). May take time for large databases.

Output example:

2026-01-17 20:21:17 [INFO] 🔍 Precise analysis mode (using COUNT queries, may take time)
...
📊 DB Capacity Planning Report
...
-------------------------------------------------------------------------------------------------------------------
 TABLE                 PERIOD           START(ACC)    END(ACC)      INCREASE      RATE        TREND
-------------------------------------------------------------------------------------------------------------------
 STAT_MEDIA_EVENT_LOG  Last 1 week      71,499        96,792        25,293        +35.4%      🟠 Rapid increase (caution)
 STAT_MEDIA_EVENT_LOG  Last 1 month     16,268        96,792        80,524        +495.0%     🔴 Surge (danger)
 ...

Key features:

  1. Accurate data: Based on actual row counts rather than InnoDB estimates
  2. Period-based trends: Calculates exact growth rates (%) by comparing with data from 1 week/1 month/1 quarter/1 year ago
  3. Risk assessment: Trend diagnosis based on growth rate (🔴 Surge, 🟠 Rapid increase, etc.)
