25. 지원 번들 (Support Bundle)

v0.6.29 추가

에어갭(Air-gapped) 환경에서 원격 진단이 불가능할 때, 시스템 상태를 한 번에 수집하여 tar.gz 파일로 내보내는 기능입니다.

25.1 개요​

사이드바에서 도구 > 지원 번들을 선택합니다. 시스템 진단 정보를 수집하여 하나의 압축 파일로 다운로드할 수 있습니다.

활용 시나리오
  • 기술 지원 요청 시 시스템 상태 첨부
  • 장애 발생 시 로그/설정/메트릭 일괄 수집
  • 정기 점검 시 시스템 스냅샷 보관

25.2 번들 생성​

번들 생성 버튼을 클릭하면 확인 다이얼로그가 나타납니다. 확인 후 수집이 시작되며 진행 상태가 실시간으로 표시됩니다.

수집 카테고리​

카테고리수집 내용저장 위치
시스템 정보호스트명, OS, 커널, 아키텍처, 업타임, 타임존system/info.json
버전 정보tkadmin, 엔진(MariaDB/Redis/Nginx/OpenSearch), TACHYON 버전system/versions.json
서비스 상태서비스별 Active 상태, PID, CPU/메모리 사용량system/services.json
시스템 로그tkadmin 로그 파일 (.log, .log.gz)logs/tkadmin/
서비스 로그주요 서비스 로그 파일 + journalctl 출력logs/services/
설정 파일tkadmin, TACHYON, nginx, MariaDB, Redis, OpenSearch 설정configs/{category}/
감사 로그최근 30일 감사 로그 (JSONL 형식, 최대 10만건)logs/tkadmin/audit.jsonl
리소스 메트릭CPU/메모리/디스크 현황 + 용량 스냅샷(90일)system/metrics.json, capacity/snapshots.jsonl

진행 상태 표시​

수집 진행 중 각 카테고리별 상태가 아이콘으로 표시됩니다:

  • 대기: 아직 수집 시작 전
  • 수집 중: 현재 데이터 수집 진행 중 (스피너)
  • 완료: 수집 성공
  • 실패: 수집 중 오류 발생

25.3 번들 이력​

생성된 번들은 이력 테이블에 표시됩니다. 각 항목에는 다음 정보가 포함됩니다:

컬럼설명
ID번들 고유 식별자 (예: sb-20260312-143500)
생성 일시번들 생성 시각
상태ready (다운로드 가능), generating (생성 중), error (실패)
크기압축 파일 크기
다운로드다운로드 버튼 (ready 상태에서만 활성)
자동 정리

서버에 최대 10개의 번들이 보관됩니다. 초과 시 가장 오래된 번들이 자동으로 삭제됩니다.

25.4 번들 다운로드​

이력 테이블에서 다운로드 버튼을 클릭하면 support-bundle-{hostname}-{timestamp}.tar.gz 파일이 다운로드됩니다.

번들 내부 구조​

support-bundle-tums4-20260312-143500.tar.gz
├── manifest.json          # 메타데이터 (호스트명, 생성시각, 카테고리 목록)
├── system/
│   ├── info.json          # 시스템 정보
│   ├── versions.json      # 버전 정보
│   ├── services.json      # 서비스 상태
│   └── metrics.json       # 리소스 메트릭
├── logs/
│   ├── tkadmin/
│   │   ├── tkadmin.log    # 시스템 로그
│   │   └── audit.jsonl    # 감사 로그
│   └── services/
│       ├── mariadb.log    # 서비스별 로그
│       └── ...
├── configs/
│   ├── tkadmin/           # tkadmin 설정
│   ├── tachyon/           # TACHYON 설정 (YML)
│   ├── nginx/             # nginx 설정
│   └── ...
└── capacity/
    └── snapshots.jsonl    # 용량 스냅샷 이력

25.5 보안 고려사항​

민감 정보 제외

번들 생성 시 다음 파일은 자동으로 제외됩니다:

  • 인증서/키 파일: .key, .crt, .pem, .p12
  • 데이터베이스 비밀번호는 설정 파일에 암호화 상태로 포함됩니다

번들 파일에는 시스템 설정과 로그가 포함되므로, 전달 시 보안에 유의하시기 바랍니다.

25.6 API 참조​

엔드포인트메서드설명
/api/support-bundle/generatePOST번들 생성 (SSE 스트림으로 진행 상태 전달)
/api/support-bundle/listGET저장된 번들 목록 조회
/api/support-bundle/download/:idGET번들 파일 다운로드
/api/support-bundle/statusGET현재 생성 중인 번들 상태 조회
