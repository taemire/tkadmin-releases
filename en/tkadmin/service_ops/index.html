18. 서비스 운영 관리

v0.6.22 추가

TACHYON 서비스의 로그 레벨, JVM 메모리, 로그 로테이션 설정을 웹 UI에서 직접 관리합니다.

18.1 개요​

기존에는 tkctl CLI를 통해서만 가능했던 서비스 운영 작업(로그 레벨 변경, JVM 튜닝, 로그 로테이션 확인)을 브라우저에서 직접 수행할 수 있습니다.

사이드바에서 인프라 > 서비스 운영을 선택하여 접근합니다.

18.2 로그 레벨 관리​

화면 구성​

컬럼설명
서비스명TACHYON 구성 서비스 이름
현재 레벨현재 설정된 로그 레벨 (DEBUG/INFO/WARN/ERROR). nginx의 경우 기본 레벨이 ERROR (Default)로 표시되며, 이는 nginx의 정상 기본 설정입니다
설정 파일로그 레벨이 정의된 설정 파일 경로
상태적용 상태

로그 레벨 변경​

  1. 변경할 서비스의 드롭다운에서 원하는 로그 레벨을 선택합니다.
  2. 확인 대화상자에서 변경을 승인합니다.
  3. 변경이 즉시 적용되며, 서비스 재시작 없이 반영됩니다.
warning

DEBUG 또는 TRACE 레벨은 로그 용량이 급격히 증가할 수 있습니다. 문제 분석 후 반드시 INFO 이상으로 복원하세요.

18.3 JVM 메모리 관리​

요약 대시보드​

상단에 시스템 전체 메모리 현황이 표시됩니다:

  • 시스템 메모리: 서버 전체 물리 메모리
  • JVM 합계 할당: 모든 Java 서비스의 Xmx 합계
  • 할당률(%): 할당 비율 (60% 미만 정상, 80% 이상 위험)

프리셋 적용​

프리셋설명
XS최소 메모리 할당 (개발/테스트 환경)
S소규모 환경 (100 에이전트 이하)
M중규모 환경 (1,000 에이전트 이하)
L대규모 환경 (1,000 에이전트 이상)

프리셋 버튼 클릭 시 모든 Java 서비스의 Xms/Xmx 값이 일괄 조정됩니다.

tip

프리셋 적용 후 서비스 재시작이 필요합니다. 서비스 관리 페이지에서 재시작하세요.

서비스별 상세​

테이블에서 각 Java 서비스의 현재 Xms, Xmx 값과 설정 파일 경로를 확인할 수 있습니다.
