20. 백업 관리

v0.6.23 추가

백업 스케줄 관리, 실행 이력 조회, 백업 파일 인벤토리를 웹 UI에서 통합 관리하고, 백업 포인트로부터 데이터를 복원합니다.

20.1 개요​

사이드바에서 스토리지 > 백업 관리를 선택합니다. 4개의 탭으로 구성됩니다:

  1. 스케줄 — 등록된 백업 스케줄 목록 및 즉시 실행
  2. 이력 — 백업 실행 결과 히스토리
  3. 인벤토리 — 디스크에 존재하는 백업 파일 목록
  4. 복원 — 백업 포인트 선택 및 데이터 복원 (v0.6.28 추가)

20.2 백업 스케줄​

컬럼설명
스케줄명백업 작업 이름
Cron 표현식실행 주기 (Cron 형식)
보관 일수백업 파일 보관 기간
마지막 상태최근 실행 결과 (성공/실패/실행중/건너뜀)
마지막 실행최근 실행 시각
즉시 실행수동 실행 버튼

즉시 실행​

즉시 실행 버튼을 클릭하면 확인 대화상자가 표시됩니다. 승인 시 해당 스케줄의 백업이 즉시 실행됩니다.

20.3 실행 이력​

컬럼설명
스케줄명실행된 백업 스케줄 이름
상태결과 (성공/실패/실행중)
시작 시각백업 시작 시점
완료 시각백업 완료 시점
출력/오류실행 로그 또는 오류 메시지

20.4 백업 인벤토리​

컬럼설명
백업 ID고유 식별자
유형백업 대상 및 방식 (mariadb/full, mariadb/incr, opensearch)
크기백업 파일 크기
검증무결성 검증 결과 (✓/✗)
생성 시각백업 파일 생성 시점
파일 경로디스크 상의 파일 위치
tip

v0.6.34부터 디스크 자동 스캔 기능이 추가되었습니다. tkctl CLI로 직접 실행한 백업이나 manifest에 미등록된 백업도 TACHYON MariaDB 백업 디렉토리({target_dir}/mariadb/backup/)를 스캔하여 자동으로 인벤토리에 표시됩니다.

20.5 데이터 복원​

v0.6.28 추가

복원 탭에서 백업 포인트를 선택하여 데이터를 복원합니다.

복원 대상​

대상설명
MariaDB 전체 복원mariabackup 기반 전체 물리 복원
MariaDB 파티션 복원특정 파티션의 논리 복원 (mysqldump 기반)
OpenSearch 스냅샷 복원스냅샷 기반 인덱스 복원

복원 절차​

  1. 복원 대상 유형을 선택합니다 (MariaDB / Partition / OpenSearch).
  2. 사용 가능한 백업 포인트 목록에서 복원할 백업을 선택합니다.
  3. 시뮬레이션 버튼으로 사전 검증을 수행할 수 있습니다 (Partition만 지원).
  4. 복원 실행 버튼을 클릭합니다.
  5. 1단계 확인: 복원 대상과 백업 포인트를 확인합니다.
  6. 2단계 확인: 최종 경고 ("이 작업은 되돌릴 수 없습니다") 후 실행합니다.
  7. 복원 완료 시 결과(성공/실패, 소요 시간)가 표시됩니다.
warning

복원은 현재 데이터를 덮어씁니다. 복원 전 반드시 현재 상태를 확인하고, 중요 데이터가 있다면 먼저 별도 백업을 수행하세요.

info

시뮬레이션 모드는 현재 MariaDB 파티션 복원에서만 지원됩니다. 시뮬레이션 시 실제 데이터는 변경되지 않습니다.

고급 복원 (수동 입력)​

v0.6.34 추가

복원 탭 하단의 "고급 복원 (수동 입력)" 접이식 섹션을 펼치면, 인벤토리에 없는 백업도 경로를 직접 입력하여 복원할 수 있습니다.

복원 유형입력 항목
MariaDB 전체백업 경로, 확인(RESTORE)
MariaDB 파티션테이블명, 파티션명, SQL 파일 경로, DryRun 체크, 확인(RESTORE)
OpenSearch 스냅샷저장소명, 스냅샷명, 확인(RESTORE)
info

이 기능은 기존 고급 운영 > 복원 탭에서 이전되었습니다. v0.6.34부터 모든 복원 기능은 백업 관리 페이지에서 통합 관리됩니다.

20.6 API 엔드포인트​

MethodPath설명
GET/api/backup/schedules백업 스케줄 목록
POST/api/backup/run/{id}스케줄 즉시 실행
GET/api/backup/history실행 이력 조회
GET/api/backup/inventory백업 파일 인벤토리
POST/api/backup/verify/{id}백업 무결성 검증 (SHA-256 체크섬, 크기 확인)
POST/api/restore/mariadbMariaDB 전체 복원 (JSON body: {full_backup_path, confirm})
POST/api/restore/mariadb/partitionMariaDB 파티션 복원 (JSON body: {table_name, partition_name, file_path, dry_run, confirm})
POST/api/restore/opensearchOpenSearch 스냅샷 복원 (JSON body: {repository, snapshot, indices, confirm})
