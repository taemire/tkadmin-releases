15. 웹 터미널

v0.6.2 — 브라우저에서 tkctl CLI를 직접 사용하는 웹 터미널

15.1 개요​

웹 터미널은 브라우저에서 직접 tkctl 대화형 CLI 쉘을 사용할 수 있는 기능입니다. 별도의 SSH 접속이나 터미널 프로그램 없이도 tkctl의 모든 명령어와 자동완성 기능을 웹 UI에서 바로 사용할 수 있습니다.

ℹ️ 웹 터미널은 서버에서 실행되는 tkctl --shell 프로세스에 PTY(가상 터미널)를 할당하여 완전한 터미널 경험을 제공합니다. go-prompt 기반의 Tab 자동완성, 명령어 히스토리 등이 모두 지원됩니다.

15.2 접근 방법​

  1. 사이드바 하단의 "도구" 섹션을 찾습니다.
  2. "터미널" 메뉴를 클릭합니다.
  3. 화면이 터미널 모드로 전환되며, 자동으로 서버에 연결됩니다.

15.3 화면 구성​

상태 표시줄​

터미널 상단에 연결 상태를 표시하는 바가 있습니다:

상태표시설명
연결됨🟢 연결됨서버와 정상 연결된 상태
연결 중🟡 연결 중...서버에 연결을 시도하는 중
연결 끊김🔴 연결 끊김서버와의 연결이 끊어진 상태
재연결 중🟡 재연결 중 (N/5)자동 재연결을 시도하는 중

상태 표시줄 우측의 새로고침 버튼을 클릭하면 수동으로 재연결할 수 있습니다.

터미널 영역​

  • tkctl> 프롬프트가 표시되면 명령어를 입력할 수 있습니다.
  • 화면 크기에 맞게 자동으로 리사이즈됩니다.
  • 다크/라이트 테마에 따라 터미널 색상이 자동으로 변경됩니다.

15.4 사용 방법​

기본 명령어​

터미널에서 다음과 같은 tkctl 명령어를 사용할 수 있습니다:

tkctl> help                    # 사용 가능한 명령어 목록 표시
tkctl> service check           # 전체 서비스 상태 확인
tkctl> service start <서비스명>  # 서비스 시작
tkctl> service stop <서비스명>   # 서비스 중지
tkctl> service restart <서비스명> # 서비스 재시작
tkctl> analyze                  # 시스템 분석
tkctl> env check               # 환경 점검
tkctl> exit                    # 터미널 종료

Tab 자동완성​

명령어 입력 중 Tab 키를 누르면 사용 가능한 명령어와 옵션이 자동완성 목록으로 표시됩니다:

tkctl> ser[Tab]
> service    서비스 관리 명령
tkctl> service [Tab]
> check      서비스 상태 확인
> start      서비스 시작
> stop       서비스 중지
> restart    서비스 재시작

특수 키​

키동작
Tab자동완성
Ctrl+C현재 입력 취소
Ctrl+D터미널 종료 (= exit)
↑ / ↓명령어 히스토리 탐색
clear화면 지우기

15.5 연결 관리​

자동 재연결​

네트워크 불안정 등으로 연결이 끊어지면 터미널이 자동으로 재연결을 시도합니다:

  • 재시도 간격: 3초
  • 최대 재시도: 5회
  • 5회 시도 후에도 연결되지 않으면 상태 표시줄에 "연결 끊김"이 표시됩니다.
  • 상태 표시줄의 새로고침 버튼을 클릭하면 재시도 횟수가 초기화되고 다시 연결을 시도합니다.

세션 관리​

  • 다른 메뉴로 이동하면 현재 터미널 세션이 자동으로 종료됩니다.
  • 터미널 메뉴로 다시 돌아오면 새 세션이 시작됩니다.
  • 서버당 최대 5개의 동시 터미널 세션이 허용됩니다.
  • 30분 동안 키 입력이 없으면 세션이 자동으로 종료됩니다.

15.6 테마 지원​

터미널은 tkadmin의 다크/라이트 테마에 자동으로 대응합니다:

  • 다크 모드: Tokyo Night 스타일 컬러 스킴 (어두운 배경 + 밝은 텍스트)
  • 라이트 모드: 밝은 배경 + 어두운 텍스트

상단 헤더의 테마 전환 버튼으로 테마를 변경하면 터미널 색상이 즉시 반영됩니다.

15.7 제한사항​

항목설명
실행 환경Linux 서버에서만 동작 (PTY 필요)
권한tkadmin 서버 프로세스와 동일한 OS 권한으로 실행
동시 세션서버당 최대 5개
유휴 타임아웃30분 무입력 시 자동 종료
브라우저 지원WebSocket을 지원하는 모든 최신 브라우저

15.8 트러블슈팅​

연결이 되지 않을 때​

  1. 서버 상태 확인: tkadmin 서비스가 정상 실행 중인지 확인합니다.
  2. tkctl 바이너리 확인: 서버에 tkctl 바이너리가 설치되어 있는지 확인합니다. 
    ls -la /usr/local/tkadmin/bin/tkctl
  3. 방화벽 확인: WebSocket 연결(포트 13700)이 방화벽에 의해 차단되지 않는지 확인합니다.
  4. 세션 한도 확인: 최대 동시 세션(5개)에 도달하지 않았는지 확인합니다.

입력이 반영되지 않을 때​

  • 상태 표시줄이 "연결됨"(녹색)인지 확인합니다.
  • 연결이 끊어졌다면 새로고침 버튼을 클릭하여 재연결합니다.
  • 브라우저 콘솔(F12)에서 WebSocket 관련 오류 메시지를 확인합니다.
