16. 쿼리마법사

16.1 개요​

  • TACHYON MariaDB와 외부 JDBC 데이터소스를 통합하여 쿼리 실행
  • TPU_POL_SYSTEM_HR에 등록된 인사DB 연결정보를 자동 읽기
  • 5종 DBMS 지원 (MariaDB, Oracle, MSSQL, PostgreSQL, DB2)
  • OpenSearch 듀얼엔진 통합: SQL Plugin + DSL 자동 판별, NDJSON 스트리밍, 가상 스크롤
  • JDBC 브릿지 아키텍처: Go 네이티브 드라이버와 Java JDBC 브릿지를 통합 인터페이스로 추상화하여 TACHYON 콘솔과 동일한 JDBC 연결 호환성 확보

16.2 접근 방법​

  • 사이드바 > 도구 > 쿼리마법사 클릭

16.3 화면 구성​

데이터소스 바​

  • 드롭다운: TACHYON MariaDB (자동 접속) + 외부 JDBC 목록 + OpenSearch
  • 연결 상태 뱃지 (Connected/Error)
  • [연결 테스트] 버튼 (외부 DB 전용)
  • OpenSearch는 서버에 연결된 경우 자동으로 목록에 표시됨

스키마 탐색기 (좌측 패널)​

  • 스키마 트리: 클릭하여 테이블 목록 펼치기
  • 테이블 클릭: 하단에 컬럼 상세 표시
  • 테이블 더블클릭: SQL 편집기에 SELECT 문 자동 생성
  • 스키마/컬럼 경계: 드래그로 크기 조절 가능
  • 스키마 패널 ↔ 쿼리 패널 리사이저: 좌측 패널 경계를 드래그하여 너비 조절 (180~600px)

테이블 Row Count 표시 (CLDR 로케일 표기)​

테이블 목록에서 각 테이블의 예상 행 수를 UI 언어에 맞는 단위로 표시합니다.

한국어 설정 (ko) — CLDR ko-KR Compact Decimal Format 표준:

표시 예의미
0빈 테이블
1,1001,100 행 (1만 미만 — 원값 표시)
~9.9만약 99,000 행 (InnoDB 추정값)
~9.5억약 95,346,335 행 (InnoDB 추정값)
1억정확히 100,000,000 행
1.2조약 1.2조 행

영어 설정 (en) — CLDR en-US Compact Decimal Format 표준:

표시 예의미
~99.3K약 99,300 행
~95.3M약 95,346,335 행

~ prefix: InnoDB 엔진의 TABLE_ROWS는 내부 통계 기반 추정값(±0.7% 오차)입니다. 정확한 행 수가 필요하면 SELECT COUNT(*) FROM 테이블명을 실행하세요.

컬럼 상세 뷰​

테이블을 클릭하면 하단에 컬럼 메타데이터가 표 형태로 표시됩니다.

컬럼설명
컬럼컬럼명 + PK/IDX 뱃지
타입데이터 타입 (예: varchar(255), timestamp)
∅Nullable 여부 — ✓ = NULL 허용, ✗ = NOT NULL (마우스 오버 시 상세 툴팁)
설명DB 컬럼 코멘트 (COLUMN_COMMENT). 마우스 오버 시 전체 텍스트 표시
  • 헤더에 테이블 설명이 있을 경우 (TABLE_COMMENT), 테이블명 하단에 서브타이틀로 표시됩니다

SQL 편집기 (우측 상단)​

  • SQL 구문 하이라이팅 (Monaco Editor)
  • Ctrl+Enter: 쿼리 실행
  • SELECT, SHOW, DESCRIBE, EXPLAIN만 허용 (안전 모드)
  • LIMIT 자동 적용 (최대 100행)
  • 에디터/결과 경계: 드래그로 크기 조절 가능

SQL 자동완성 및 구문 강조​

MariaDB 전용 SQL 토크나이저와 지능형 자동완성을 제공합니다.

기능설명
Monarch 토크나이저120+ SQL 키워드, 60 MariaDB 내장 함수, 대소문자 무시
AFTER_DOT 자동완성schema.table.col 3단계 한정자 체인 — 도트 입력 시 다음 단계 후보 목록 표시
AFTER_USE 자동완성USE <DBNAME>; 입력 후 스키마 목록 자동 제안
KEYWORD 자동완성SQL 키워드·함수 맥락 감지 자동 제안
테마 자동 동기화라이트/다크 모드 전환 시 에디터 테마 즉시 반영 (중앙 Theme 서비스 구독)
USE DBNAME; 처리클라이언트 전용 처리 — 스키마 탐색기 자동 동기화

Monaco 스마트 지수 — 5종 Provider​

Monaco Editor는 5종 스마트 Provider를 지원합니다.

SQL 함수 위에 마우스를 올리면 MariaDB 레퍼런스 문서가 인라인으로 표시됩니다.

40+ 내장 함수 지원:

카테고리함수 예시

SignatureHelp — 파라미터 힌트​

함수 괄호 안에서 Ctrl+Shift+Space를 누르면 파라미터 도움말이 표시됩니다. 콤마 입력 시 현재 파라미터가 자동으로 강조됩니다.

SQL Lint — 위험 쿼리 실시간 경고​

쿼리 입력 후 600ms 내에 자동으로 정적 분석을 수행하여 위험 패턴을 표시합니다.

규칙심각도조건메시지
R1⚠️ WarningSELECT * + LIMIT 없음"SELECT * 사용 시 LIMIT을 지정하지 않으면 전체 데이터를 로드합니다."
R2❌ ErrorDELETE/UPDATE + WHERE 없음"DELETE/UPDATE 에 WHERE 조건이 없습니다. 전체 데이터 삭제/수정이 발생할 수 있습니다."
R3ℹ️ Info세미콜론 누락"쿼리 끝에 세미콜론(;)이 없습니다."
danger

R2 Error: WHERE 없이 DELETE FROM 테이블명 또는 UPDATE 테이블명 SET ...을 입력하면 빨간 물결 밑줄과 에러 마커가 즉시 표시됩니다. 실행 전 반드시 WHERE 조건을 추가하십시오.

FoldingRange — 코드 블록 접기​

-- region / -- endregion 주석 패턴으로 SQL 블록을 접을 수 있습니다.

-- region 조회 쿼리
SELECT id, name, email
FROM users
WHERE active = 1
-- endregion

-- region 집계 쿼리
SELECT COUNT(*) as cnt, DATE(created_at) as dt
FROM orders
GROUP BY DATE(created_at)
-- endregion
  • Ctrl+Shift+[ : 현재 블록 접기
  • Ctrl+Shift+] : 현재 블록 펼치기
  • Ctrl+K, Ctrl+0 : 모든 블록 접기
  • Ctrl+K, Ctrl+J : 모든 블록 펼치기
괄호 자동완성 및 주석 토글​
기능동작
괄호 자동완성( 입력 시 ) 자동 삽입, ' 입력 시 닫는 따옴표 자동 삽입
주석 토글Ctrl+/ — 선택 범위에 -- 주석 일괄 추가/제거
스니펫 자동완성 체인 (Snippet Autocomplete Chain)​

DML 스니펫 삽입 후 Tab 키로 다음 플레이스홀더로 이동할 때 자동완성 목록이 자동으로 트리거됩니다.

동작 방식:

  1. 스니펫 삽입 (예: SELECT ... FROM ... WHERE ... 템플릿)
  2. Tab 키로 다음 플레이스홀더로 이동
  3. 이동 즉시 해당 위치의 컨텍스트 자동완성 목록 자동 표시 (50ms 내)
  4. 화살표 키나 입력으로 원하는 항목 선택 후 계속 진행

지원 스니펫 예시:

스니펫 트리거확장 결과
selSELECT * FROM <table>
selwSELECT * FROM <table> WHERE <condition>
cntSELECT COUNT(*) FROM <table>
grpSELECT <col>, COUNT(*) FROM <table> GROUP BY <col>
tip

스니펫 자동완성 체인을 사용하면 테이블명, 컬럼명을 손으로 타이핑하지 않고 Tab → 선택 → Tab → 선택 흐름으로 빠르게 쿼리를 완성할 수 있습니다.

결과 패널 (우측 하단)​

  • 컬럼 헤더 + 데이터 테이블
  • 실행 시간, 행 수, 잘림 여부 표시
  • CSV/JSON 내보내기
  • IndexNotFoundException UX: OpenSearch에서 존재하지 않는 인덱스 쿼리 시 가용 인덱스 목록을 배지(badge)로 표시 — 클릭 시 SQL 편집기에 인덱스명 자동 삽입

히스토리​

  • 우측 슬라이드 패널 (Ctrl+H 또는 히스토리 버튼)
  • 최근 실행 쿼리 50건 (실행 시각은 대시보드 시각 표시 기준 설정에 따라 표시됨)
  • 즐겨찾기 (별표) 기능
  • 클릭하여 편집기에 복원

JDBC 브릿지 연결​

쿼리마법사는 JDBC 브릿지를 통해 TACHYON 콘솔과 동일한 JDBC 연결 호환성을 제공합니다.

Go (tkadmin) ─── DBConnector 인터페이스 ──┬── NativeConnector (Go 네이티브 드라이버)
                                          └── JDBCConnector  (Java subprocess TCP JSON-RPC)
항목설명
연결 전략Go 네이티브 드라이버 우선, 실패 시 JDBC 브릿지 자동 폴백
DB2 지원Go에 DB2 네이티브 드라이버가 없으므로 JDBC 브릿지 전용
TACHYON JRE 재활용서버에 설치된 TACHYON JRE(/usr/local/TACHYON/TTS40/java/)와 JDBC 드라이버를 그대로 사용
프로토콜TCP localhost JSON-RPC (4-byte length prefix + JSON)
Java subprocess최초 연결 시 자동 시작, 서버 종료 시 자동 정리
JDBC 브릿지 요구사항

JDBC 브릿지를 사용하려면 서버에 Java 11 이상이 설치되어 있어야 합니다. TACHYON이 정상 설치된 환경에서는 내장 JRE가 자동으로 감지됩니다. Java가 설치되지 않은 환경에서는 Go 네이티브 드라이버만 사용됩니다.

DB2 연결

DB2 데이터베이스에 연결하려면 TACHYON 서버에 DB2 JDBC 드라이버(db2jcc*.jar)가 /usr/local/TACHYON/TTS40/api/libs/ 또는 /usr/local/TACHYON/TTS40/libs/ 경로에 배치되어 있어야 합니다.

16.5 OpenSearch 모드​

OpenSearch 데이터소스를 선택하면 전용 인터페이스가 활성화됩니다.

SQL / DSL 쿼리 탭​

  • SQL 탭: OpenSearch SQL Plugin을 통한 SQL 쿼리 실행
  • DSL 탭: JSON 기반 Query DSL 직접 작성 (Monaco JSON 에디터)

DSL 템플릿​

DSL 모드에서 8종 템플릿을 드롭다운으로 선택하여 빠르게 쿼리를 시작할 수 있습니다:

템플릿설명
match_all전체 문서 조회
match특정 필드 텍스트 검색
range날짜/숫자 범위 검색
bool복합 조건 (must/should/must_not)
date_histogram날짜 기반 히스토그램 집계
terms필드별 그룹 집계
stats통계 (min/max/avg/sum)
anomaly이상 탐지 쿼리

인덱스 브라우저​

  • 스키마 패널에 인덱스 패턴 목록 표시 (예: tachyon_media_*)
  • 패턴 클릭 시 개별 인덱스 펼치기 (문서 수 표시)
  • 인덱스 클릭 시 필드 목록 (타입, Nullable 정보)

16.6 대용량 쿼리 기능​

NDJSON 스트리밍​

  • MaxRows를 "무제한" 또는 10,000행 이상으로 설정하면 자동으로 스트리밍 모드 전환
  • 서버 메모리를 거의 사용하지 않고(O(1)) 대량 행을 브라우저로 전송
  • 진행률 표시: 수신된 행 수, 전송 속도(rows/sec)
  • [취소] 버튼으로 실행 중인 스트리밍 즉시 중단
  • 스트리밍 타임아웃: 최대 2시간 (100M 행 이상 풀스캔 지원)

가상 스크롤 (VirtualTable)​

  • 수만~수십만 행을 화면에 렌더링해도 브라우저 성능 저하 없음
  • 화면에 보이는 영역만 DOM 렌더 (나머지는 가상 높이 유지)
  • 스크롤 시 부드러운 행 전환
  • 표시 한계: 최대 50만 행 화면 표시. 초과 시 자동 완료 처리 후 상태바에 미표시 행 수 안내

커서 페이지네이션​

  • MaxRows 드롭다운에서 "페이지네이션" 선택
  • 페이지당 10,000행씩 탐색 ([이전] / [다음] 버튼)
  • 전체 행 수 추정값 표시 (EXPLAIN 기반)

안전장치​

상황동작
10만행 초과 쿼리확인 다이얼로그 표시
100만행 초과자동으로 페이지네이션 모드 제안
메모리 부족브라우저 메모리 사용량 경고
동시 스트리밍 3개 초과대기 (서버측 세마포어)
표시 50만행 한계 도달자동 완료 처리 + 서버 스트림 취소 (브라우저 Freeze 방지)
내보내기 10만행 한계초과 행은 내보내기 배열에서 제외 (GC Stall 방지)
100M 행 이상 쿼리 시 예상 동작
  1. 스트리밍 시작 → VirtualTable에 실시간 렌더링
  2. 50만 행 도달 시 자동으로 완료 처리 (서버 쿼리 취소)
  3. 상태바 (ko): 완료 (Ns) 50만행 한계 — 나머지 N행 미표시
  4. 화면에는 50만 행이 가상 스크롤로 표시됨
  5. CSV/JSON 내보내기는 최대 10만 행까지 지원

페이지 이탈 시 처리​

스트리밍 도중 다른 메뉴로 이동하면 자동으로 정리됩니다:

  • 브라우저: fetch 스트림 즉시 취소 (AbortController)
  • 서버: HTTP 연결 끊김 감지 → MariaDB 쿼리 자동 KILL
  • 세션 상태 자동 저장 (복귀 시 이전 스키마/쿼리 복원)
