23. 고급 운영

관리자 계정 관리, UI 주입 복구, 서비스 업데이트, 셀프 업데이트, 데이터베이스 복원 등 고급 운영 작업을 수행합니다.

23.1 개요​

사이드바에서 운영 > 고급 운영을 선택합니다. 5개 탭으로 구성됩니다:

  1. 관리자 관리 — 사용자 비밀번호 초기화
  2. UI 주입 복구 — TACHYON 대시보드 UI 인젝션 복구
  3. 서비스 업데이트 — 서비스 패키지 업데이트
  4. 셀프 업데이트 — tkadmin 자체 업데이트 (온라인 / 오프라인)
  5. 복원 — MariaDB / OpenSearch 데이터 복원

23.2 관리자 관리​

TACHYON 관리자 계정을 조회하고 비밀번호를 초기화합니다.

컬럼설명
사용자 ID관리자 계정 ID
사용자명관리자 이름
테넌트 DB소속 데이터베이스
비밀번호 초기화초기화 버튼

비밀번호 초기화 버튼 클릭 시 확인 대화상자가 표시됩니다. 승인하면 해당 계정의 비밀번호가 기본값으로 초기화됩니다.

23.3 UI 주입 복구​

TACHYON SPA 대시보드에 tkadmin 링크가 표시되지 않는 경우, UI 인젝션을 복구합니다.

  • 복구 버튼을 클릭하면 자동으로 인젝션 상태를 점검하고 복구합니다.
  • 결과: Repaired (복구됨) 또는 Intact (정상, 변경 불필요)

23.4 서비스 업데이트​

개별 서비스의 패키지를 업데이트합니다.

  1. 서비스 선택: Nginx, Redis, OpenSearch, MariaDB 중 선택
  2. 파일 경로 입력: 업데이트 패키지 파일 경로 (예: /tmp/package.tar.gz)
  3. 강제 업데이트: 버전 체크를 건너뛰고 강제 적용 (선택)
  4. 업데이트 실행: 확인 대화상자 승인 후 실행
경고

서비스 업데이트 중 해당 서비스가 일시 중단될 수 있습니다. 운영 시간 외에 수행하세요.

23.5 셀프 업데이트​

tkadmin 바이너리 자체를 업데이트합니다. 온라인 모드와 오프라인(에어갭) 모드 두 가지를 지원합니다.

온라인 업데이트 (GitHub Releases)​

인터넷 연결이 가능한 환경에서 GitHub Releases를 통해 최신 버전을 자동 다운로드 및 적용합니다.

  1. 업데이트 확인 버튼 클릭 → 현재 버전과 최신 릴리즈 버전 비교
  2. 신규 버전이 있으면 다운로드 및 적용 버튼 활성화
  3. 진행 상황을 SSE 스트리밍으로 실시간 표시
  4. 완료 후 서비스 자동 재시작
정보

update_mirror 설정 필드로 커스텀 미러 서버를 지정할 수 있습니다. 빈 값이면 GitHub 기본 경로를 사용하고, "disabled" 설정 시 온라인 업데이트가 비활성화됩니다.

오프라인 업데이트 (파일 업로드)​

인터넷 연결이 불가한 에어갭(Air-Gap) 환경에서 사전 다운로드한 패키지로 업데이트합니다.

  1. .zip 또는 .tar.gz 아카이브 파일을 영역에 드래그 앤 드롭 (또는 클릭 선택)
  2. 아카이브에서 tkadmin 바이너리를 자동 탐색 및 추출 (매직 바이트 자동 감지)
  3. 버전 비교 결과 확인 후 업데이트 적용 버튼 클릭
경고

업데이트 적용 중 tkadmin 서비스가 재시작됩니다. 이 시간 동안 잠시 접속이 끊길 수 있습니다. (systemd가 자동 재시작하므로 수동 조치는 불필요합니다.)

팁

업데이트 전 설정 백업을 권장합니다. 백업은 백업 관리 메뉴에서 수행하세요.

23.6 복원​

위험

복원 작업은 되돌릴 수 없습니다. 실행 전 반드시 현재 데이터의 백업 여부를 확인하세요.

MariaDB 전체 복원​

전체 MariaDB 데이터베이스를 백업 파일로부터 복원합니다.

  1. 백업 파일 경로를 입력합니다.
  2. 확인란에 RESTORE를 입력합니다.
  3. 복원 실행 버튼을 클릭합니다.

MariaDB 파티션 복원​

특정 테이블의 파티션만 선택적으로 복원합니다.

  1. 테이블명, 파티션명, 백업 파일 경로를 입력합니다.
  2. Dry-run 체크 시 실제 복원 없이 시뮬레이션만 수행합니다.
  3. Dry-run 미사용 시 RESTORE 확인 입력이 필요합니다.

OpenSearch 스냅샷 복원​

OpenSearch 스냅샷 리포지터리에서 데이터를 복원합니다.

  1. 리포지터리명, 스냅샷명을 입력합니다.
  2. 확인란에 RESTORE를 입력합니다.
  3. 복원 실행 버튼을 클릭합니다.
